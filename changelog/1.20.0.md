### Breaking changes ‚ùó

There are no breaking changes in 1.20.0.

### Features ‚ú®

- web: A site-wide workspace template policy may now be managed at **Manage >
  Admin > Templates > Template Policy**. If not set, a sensible default is
  provided.
- web: The _nodeSelectors_ field was added to workspace templates
- other: There is a new JsonSchema for writing Coder workspaces as code template
  with code completion
- web: Added a service message that's displayed to all users of the system. The
  message may be dismissed by each user at any point and is not shown again
  until there is a new message.
- web: System banners can now wrap text
- infra: TLS management (/_TODO_/)
- web: Opt-in for HTTP Strict Transport Security was added. This setting can be
  managed at **Manage > Admin > Infrastructure > HTTP Strict Transport
  Security**
- web: Opt-in for Secure Cookies was added. This setting can be managed at
  **Manage > Admin > Infrastructure > Secure Cookie**
- templates: Kubernetes annotations are now available in workspace templates
- infra: Increased network performance for creating new workspaces
- infra: A `CODER_RUNTIME` environment variable was added
- web: Decomissioned workspaces (awaiting deletion) are now visible in the UI

### Bug fixes üêõ

- web: Fixed a bug that caused duplicate fetch requests on page load
- web: Fixed an issue that caused a private dev URL to load as a blank page for
  other users. Other usess now see a meaningful error

### Security updates üîê

- web: Previously, information about other users was visible in payloads for
  workspace images. This information has been remove; users can now only see
  their own data
- web: Content security policy (CSP) headers were added to secure loaded and
  executed content
- infra: Upgraded Red Hat UBI from 8.3 to 8.4
- web: Client-side token generation now uses crypto APIs instead of string APIs.
- web: CVE-2021-23364 in browserslist was resolved
- web: CVE-2021-23358 in underscore was resolved
- web: CVE-2020-7753 in trim was resolved
- web: The audit log can now be filtered by the _auto-off_ action
- other: `gosec` was added as a required check in our development process. All
  flagged results were reviewed and addressed.
- other: `SonarCloud` static code analysis was added as a required check in our
  development process. All flagged results were reviewed and addressed.
